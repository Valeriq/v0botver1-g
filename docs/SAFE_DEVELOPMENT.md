# Протокол безопасной разработки

## 🎯 Цель

Минимизировать риск поломки сайта после изменений.

---

## 📋 Правила для AI

### Перед началом работы:

1. **Всегда проверять сервер** перед изменениями:
   ```bash
   curl -s -o /dev/null -w "%{http_code}" http://localhost:5173
   ```
   Если возвращает 200 — сайт работает.

2. **Спрашивать разрешение** перед рискованными изменениями:
   - Изменение более 3 файлов
   - Удаление кода
   - Изменение конфигов (vite, tailwind, tsconfig)
   - Установка новых зависимостей

3. **Сохранять в git** перед крупными изменениями:
   ```bash
   git add .
   git commit -m "checkpoint before changes"
   ```

---

### После изменений:

1. **Проверить компиляцию** — сервер должен перезапуститься без ошибок
2. **Сказать пользователю**: "Проверь в браузере: http://localhost:5173"
3. **Ждать подтверждения** что сайт работает
4. **Только после подтверждения** — продолжать или сохранять изменения

## 📝 Шаблон задачи для пользователя

```
[страница/компонент]: [что сделать]

Примеры:
- Dashboard: изменить заголовок на "Моя панель"
- Contacts: добавить фильтр по статусу
- Footer: убрать ссылку на Facebook
```

---

## ⚠️ Что считается рискованным

| Действие | Риск | Что делать |
|----------|------|------------|
| Изменение 1-2 файлов | Низкий | Можно делать сразу |
| Изменение 3+ файлов | Средний | Спросить разрешение |
| Изменение конфигов | Высокий | Обязательно спросить |
| Удаление файлов | Высокий | Спросить + сделать backup |
| Установка npm пакетов | Высокий | Спросить + проверить после |

---

## 🔄 Если сайт сломался

1. **Не паниковать**
2. **Проверить ошибки в консоли** браузера (F12)
3. **Откатить изменения**:
   ```bash
   git checkout .
   git clean -fd
   ```
4. **Переустановить зависимости**:
   ```powershell
   cd client
   pnpm install
   pnpm dev
   ```
