# Надёжная система запуска и восстановления

## Обзор

Этот документ описывает новую систему для запуска веб-приложения с автоматическим восстановлением при ошибках.

## Проблема

Раньше при падении сервера нужно было вручную перезапускать приложение. Это было неудобно при разработке и тестировании.

## Решение

Реализована многоуровневая система защиты:

### 1. **Graceful Shutdown (сервер)**

Сервер корректно завершает работу при получении сигналов:

- `SIGTERM` - от операционной системы
- `SIGINT` - от пользователя (Ctrl+C)
- `uncaughtException` - при необработанных исключениях

**Файл:** `server/index.ts`

```typescript
// Graceful shutdown автоматически:
// - Закрывает HTTP сервер
// - Логирует все события
// - Принудительно завершает через 10 сек
```

### 2. **Error Boundary (React)**

Клиентская часть обрабатывает ошибки React:

- Поймает ошибки в компонентах
- Показывает дружелюбное сообщение
- Сохраняет логи в localStorage
- Предлагает перезагрузку страницы

**Файл:** `client/src/components/ErrorBoundary.tsx`

### 3. **Автоматический перезапуск (nodemon)**

Следит за файлами и перезапускает сервер при изменениях:

- Игнорирует тестовые файлы
- Задержка 1 сек для стабильности
- Поддерживает горячую перезагрузку

**Файл:** `nodemon.json`

### 4. **Скрипт надёжного запуска**

Постоянно следит за сервером и перезапускает его при падении:

**Файл:** `scripts/dev-server.js`

Функции:
- ✅ Автоматический перезапуск через 3 сек после падения
- ✅ Логирование всех ошибок в `logs/errors.log`
- ✅ Обработка всех сигналов завершения
- ✅ Подробные сообщения о состоянии

## Команды запуска

### Рекомендуемый способ (наиболее надёжный)

```bash
npm run dev:safe
```

**Преимущества:**
- Автоматический перезапуск при ошибках
- Подробное логирование
- Работает как в разработке, так и в продакшене

### Стандартный способ (с nodemon)

```bash
npm run dev
```

**Преимущества:**
- Быстрый запуск
- Горячая перезагрузка файлов
- Подходит для активной разработки

### Прямой запуск (без защиты)

```bash
npm run dev:legacy
```

**Использование:**
- Для отладки
- Когда нужен полный контроль над процессом

## Логирование

### Логи сервера

Все ошибки логируются в:
- Консоль (stdout/stderr)
- Файл `logs/errors.log`

Формат лога:
```
[2026-01-21T09:59:45.061Z] [INFO] 📋 Скрипт надёжного запуска инициализирован
[2026-01-21T09:59:45.064Z] [ERROR] ❌ Ошибка запуска сервера: ...
```

### Логи клиента

Ошибки React сохраняются в `localStorage`:
```javascript
// Просмотр последних ошибок
JSON.parse(localStorage.getItem('errorLogs') || '[]')
```

## Проверка работоспособности

### 1. Проверка graceful shutdown

```bash
# Запустите сервер
npm run dev:safe

# Отправьте SIGTERM (или нажмите Ctrl+C)
# Сервер должен корректно завершиться
```

### 2. Проверка Error Boundary

```javascript
// Добавьте в компонент для тестирования
throw new Error("Тестовая ошибка");
```

Ожидается:
- Показывается страница с ошибкой
- Ошибка логируется в localStorage
- Кнопка "Обновить страницу" работает

### 3. Проверка автоперезапуска

```bash
npm run dev:safe

# В другом терминале убейте процесс сервера
# Например: pkill -f "server/index.js"
```

Ожидается:
- Сервер автоматически перезапускается через 3 сек
- Логируется причина падения
- Приложение продолжает работать

## Устранение неполадок

### Сервер падает бесконечно

**Причина:** Критическая ошибка в коде

**Решение:**
1. Проверьте `logs/errors.log`
2. Найдите последнюю ошибку
3. Исправьте проблему в коде
4. Перезапустите сервер

### Error Boundary не ловит ошибки

**Причина:** Ошибка в асинхронном коде

**Решение:**
```javascript
// Используйте try/catch в асинхронном коде
try {
  await someAsyncOperation();
} catch (error) {
  console.error(error);
  // Обработка ошибки
}
```

### Нодемон не перезапускается

**Причина:** Изменения в игнорируемых файлах

**Решение:**
- Проверьте `nodemon.json`
- Убедитесь, что файлы не в списке `ignore`

## Мониторинг

### Просмотр логов в реальном времени

```bash
# Linux/Mac
tail -f logs/errors.log

# Windows (PowerShell)
Get-Content logs/errors.log -Wait
```

### Анализ падений

```bash
# Посчитать количество падений
grep "Сервер упал" logs/errors.log | wc -l

# Показать последние 10 ошибок
tail -n 10 logs/errors.log
```
