# Деплой на Render.com

Пошаговая инструкция по развертыванию AI Cold Email Bot на Render.com.

## Шаг 1: Подготовка

Убедитесь, что ваш проект на GitHub и все файлы закоммичены:
```bash
git add .
git commit -m "Prepare for Render deployment"
git push origin main
```

## Шаг 2: Создайте аккаунт на Render

1. Откройте https://render.com
2. Нажмите **"Get Started"** или **"Sign Up"**
3. Выберите **"Sign up with GitHub"**
4. Авторизуйте Render доступ к вашим репозиториям

## Шаг 3: Создайте Blueprint

Blueprint - это способ развернуть сразу все сервисы из одного файла конфигурации.

1. В дашборде Render нажмите **"New +"** → **"Blueprint"**
2. Выберите **"Connect a repository"**
3. Найдите репозиторий `v0-ai-cold-email-bot` и нажмите **"Connect"**
4. Render автоматически найдет файл `render.yaml`
5. Нажмите **"Apply"**

## Шаг 4: Настройте Environment Variables

После создания Blueprint нужно добавить секретные ключи:

### Для сервиса `telegram-bot`:
- `TELEGRAM_BOT_TOKEN` = `7631533721:AAHgroaH4CGTpjraoJxUlYLAFsR3Q-uSuqM`

### Для сервиса `ai-orchestrator`:
- `OPENAI_API_KEY` = ваш ключ OpenAI (получите на https://platform.openai.com)

### Для сервиса `gmail-service` (опционально, для полного функционала):
- `GOOGLE_CLIENT_ID` = из Google Cloud Console
- `GOOGLE_CLIENT_SECRET` = из Google Cloud Console
- `GOOGLE_REDIRECT_URI` = `https://gmail-service.onrender.com/oauth/callback`
- `GMAIL_PUBSUB_TOPIC` = из Google Cloud Console

**Как добавить переменные:**
1. Откройте каждый сервис в дашборде
2. Перейдите в раздел **"Environment"**
3. Нажмите **"Add Environment Variable"**
4. Введите ключ и значение
5. Нажмите **"Save Changes"**

## Шаг 5: Запустите миграции базы данных

После того как сервис `core-api` запустится:

1. Откройте сервис `core-api` в дашборде
2. Перейдите в **"Shell"**
3. Выполните команду:
```bash
npm run migrate
```

Это создаст все необходимые таблицы в PostgreSQL.

## Шаг 6: Протестируйте бота

1. Откройте Telegram
2. Найдите вашего бота
3. Отправьте `/start`
4. Должно появиться главное меню

## Ограничения бесплатного плана Render

- Сервисы "засыпают" после 15 минут неактивности
- При первом запросе после сна нужно ~30 секунд на пробуждение
- PostgreSQL: 1GB диск
- Redis: 25MB памяти
- Достаточно для тестирования, но не для production

## Обновление проекта

Когда вы или v0 делаете изменения в коде:

1. Изменения автоматически пушатся в GitHub
2. Render автоматически обнаруживает изменения
3. Автоматически пересобирает и деплоит сервисы
4. Обычно занимает 3-5 минут

## Просмотр логов

Чтобы увидеть, что происходит в системе:

1. Откройте любой сервис в дашборде
2. Перейдите в раздел **"Logs"**
3. Вы увидите real-time логи с request-id и job-id

## Troubleshooting

### Бот не отвечает
- Проверьте логи сервиса `telegram-bot`
- Убедитесь, что `TELEGRAM_BOT_TOKEN` правильный
- Проверьте, что `CORE_API_URL` доступен

### Ошибка подключения к базе
- Дождитесь полного запуска PostgreSQL (может занять 1-2 минуты)
- Убедитесь, что миграции выполнены

### AI не генерирует письма
- Проверьте, что `OPENAI_API_KEY` добавлен в `ai-orchestrator`
- Проверьте баланс на аккаунте OpenAI
