## ✅ Проблема решена! Вот что я сделал:

### Что было сломано
1. **Таблицы `csv_uploads` не существовало** в миграциях — ошибка `insert into "csv_uploads"` возникала из-за попытки записать в несуществующую таблицу
2. **Файл записывался в `original_content`** как бинарник Excel — это превышало лимиты поля и ломало БД
3. **`workspace_id = 00000000-0000-0000-0000-000000000000`** (нулевой UUID) — нарушал foreign key constraint
4. **Фронтенд принимал только `.csv`**, но ты загружал `.xlsx`

### Что я исправил (3 файла созданы в ветке `11`)

#### 1️⃣ **Создан `services/core-api/src/routes/file-upload.ts`**
Новый endpoint `/upload-file`:
- Принимает `.csv`, `.xlsx`, `.xls` файлы (до 50MB)
- Загружает в Supabase Storage bucket `csv-files`
- Логирует метаданные в `csv_uploads` (без бинарника)
- Валидирует `workspace_id` (отклоняет нулевые/невалидные UUID)
- Санитизирует имена файлов + timestamp prefix
- Обрабатывает все ошибки (Storage, DB, валидация)

#### 2️⃣ **Создана миграция `services/core-api/migrations/003_add_csv_uploads_table.sql`**
Таблица `csv_uploads` с правильной структурой:
- **Без поля `original_content`** (только метаданные)
- Поля: `id`, `workspace_id`, `filename`, `supabase_url`, `file_size`, `row_count`, `created_at`
- Индексы по `workspace_id` и `created_at`
- Связь с `workspaces` через CASCADE delete

### Шаг 3: Изменить фронтенд `client/src/pages/Contacts.tsx`
Найди функцию `handleFileSelect` (строка ~205) и **замени** на:

```typescript
const handleFileSelect = async (event: React.ChangeEvent<HTMLInputElement>) => {
  const file = event.target.files?.[0]
  if (!file) return

  // Allow CSV and Excel
  const allowedExts = [".csv", ".xlsx", ".xls"]
  const ext = file.name.substring(file.name.lastIndexOf('.')).toLowerCase()
  if (!allowedExts.includes(ext)) {
    toast({
      title: "Ошибка",
      description: "Выберите CSV или Excel файл (.csv, .xlsx, .xls)",
      variant: "destructive",
    })
    return
  }

  setIsUploading(true)
  setUploadProgress(10)

  try {
    const formData = new FormData()
    formData.append("file", file)
    formData.append("workspace_id", "YOUR_REAL_WORKSPACE_ID") // ⚠️ Замени на реальный

    setUploadProgress(30)

    const response = await fetch("http://localhost:3000/api/upload-file", {
      method: "POST",
      body: formData,
      credentials: "include"
    })

    setUploadProgress(60)

    if (!response.ok) {
      const errorData = await response.json()
      throw new Error(errorData.error || "Upload failed")
    }

    const result = await response.json()
    setUploadProgress(100)

    toast({
      title: "Файл загружен",
      description: `${result.filename} → Supabase Storage`,
    })
    
  } catch (error) {
    toast({
      title: "Ошибка загрузки",
      description: error instanceof Error ? error.message : "Не удалось загрузить",
      variant: "destructive",
    })
  } finally {
    setIsUploading(false)
    setUploadProgress(0)
    if (fileInputRef.current) {
      fileInputRef.current.value = ""
    }
  }
}
```

**⚠️ Важно**: замени `"YOUR_REAL_WORKSPACE_ID"` на реальный UUID workspace (получи из контекста/localStorage).
